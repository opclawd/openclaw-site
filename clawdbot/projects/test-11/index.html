Test #11

Site Watchdog

Script de monitoreo de disponibilidad web

Consigna

Crear y ejecutar un script Bash (scripts/site_watchdog.sh) que:

  • Use curl para obtener el HTTP status code de https://example.com
  • Si el código no es 200, incremente el contador de fallos en alerts/fail_count.txt
  • Si hay 3 o más fallos consecutivos, cree el archivo alerts/site_down.txt con la alerta
  • Test: El sitio debería responder con código 200

Pasos Técnicos

1. Crear el script scripts/site_watchdog.sh:

#!/bin/bash

URL="https://example.com"
STATUS_CODE=$(curl -s -o /dev/null -w "%{http_code}" "$URL")

mkdir -p alerts

if [ "$STATUS_CODE" -eq 200 ]; then
    echo "✓ Site is UP (HTTP $STATUS_CODE)"
    if [ -f alerts/fail_count.txt ]; then
        rm alerts/fail_count.txt
        echo "Fail count reset."
    fi
else
    echo "✗ Site is DOWN (HTTP $STATUS_CODE)"
    
    if [ -f alerts/fail_count.txt ]; then
        FAIL_COUNT=$(cat alerts/fail_count.txt)
        FAIL_COUNT=$((FAIL_COUNT + 1))
    else
        FAIL_COUNT=1
    fi
    
    echo "$FAIL_COUNT" > alerts/fail_count.txt
    
    if [ "$FAIL_COUNT" -ge 3 ]; then
        echo "ALERT: Site has been down for $FAIL_COUNT consecutive checks." > alerts/site_down.txt
        echo "$(date): Site DOWN - HTTP $STATUS_CODE" >> alerts/site_down.txt
    fi
fi

2. Dar permisos de ejecución:

chmod +x scripts/site_watchdog.sh

3. Ejecutar el script:

./scripts/site_watchdog.sh

Resultado

Test EXITOSO

El script se ejecutó correctamente y el sitio respondió con código HTTP 200.

Salida del script:

✓ Site is UP (HTTP 200)

Detalles:

  • URL monitoreada: https://example.com
  • Estado: HTTP 200 OK
  • Archivo de alertas: No creado (sitio funcionando)
  • Contador de fallos: No incrementado (sitio UP)

Notas

  • El script usa curl -s -o /dev/null -w "%{http_code}" para obtener solo el código de estado HTTP
  • Si el sitio está caído, el contador se incrementa en alerts/fail_count.txt
  • Después de 3 fallos consecutivos, se genera la alerta alerts/site_down.txt
  • Si el sitio vuelve a estar disponible (HTTP 200), el contador se resetea automáticamente
